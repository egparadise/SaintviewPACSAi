# 02. AI 에이전트형 PACS — 아키텍처 & 하네스

> saintvidw pacs ai를 **LLM 에이전트가 PACS 기능을 수행**하는 구조(Claude Agent SDK 스타일)로 설계한다.
> 본 문서는 5대 빌딩블록 중 **하네스(Harness)** 와 **에이전트(Agents)** 를 정의한다.
> 스킬·훅·프롬프트는 `03_스킬카탈로그.md`, `03b_훅_가드레일.md`, `03c_프롬프트_라이브러리.md` 참조.

---

## 0. 5대 빌딩블록 개념 정의

| 블록 | 역할 | PACS 맥락 예 |
|------|------|--------------|
| **프롬프트(Prompt)** | 에이전트의 정체성·정책·태스크 지시 | "너는 판독 보조 에이전트다…" |
| **스킬(Skill)** | 재사용 가능한 단위 기능(도구+절차+지식) | `dicom_query`, `measure_ctr`, `report_draft` |
| **훅(Hook)** | 수명주기 가로채기(검증·감사·가드레일) | PreToolUse PHI 차단, PostToolUse 감사로그 |
| **하네스(Harness)** | 에이전트 실행 런타임(루프·도구라우팅·컨텍스트·권한) | 세션 매니저, DICOM 데이터 액세스 계층 |
| **에이전트(Agent)** | 특정 도메인을 담당하는 자율 단위 | Worklist/Viewer/Reporting/Transfer 에이전트 |

## 1. 하네스(Harness) — 런타임 구조

```
┌──────────────────────────────────────────────────────────────┐
│                    saintvidw pacs ai Harness                  │
│                                                                │
│  ┌────────────┐   ┌──────────────┐   ┌────────────────────┐  │
│  │ Session &  │   │ Orchestrator │   │  Tool Router        │  │
│  │ Context Mgr│──▶│   Agent      │──▶│ (skill dispatch)    │  │
│  └────────────┘   └──────────────┘   └─────────┬──────────┘  │
│        │                  │                     │             │
│  ┌─────▼──────┐    ┌──────▼───────┐     ┌───────▼─────────┐   │
│  │ Hook Engine│    │ Sub-Agents   │     │ Skill Registry  │   │
│  │ (pre/post) │    │ (도메인별)    │     │ (capabilities)  │   │
│  └─────┬──────┘    └──────────────┘     └───────┬─────────┘   │
│        │                                         │            │
│  ┌─────▼─────────────────────────────────────────▼────────┐  │
│  │       Data Access Layer (의료영상·데이터 추상화)          │  │
│  │  DICOMweb(QIDO/WADO/STOW) · DIMSE(C-FIND/MOVE/STORE)     │  │
│  │  Report DB · User Prefs · Audit Store · AI Model Gateway │  │
│  └──────────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────┘
```

### 1.1 하네스 책임
1. **세션·컨텍스트 관리**: 로그인 사용자, 현재 환자/검사/시리즈, 활성 레이아웃, 사용자 설정(prefs)·행잉프로토콜을 컨텍스트로 유지.
2. **오케스트레이션 루프**: 사용자 의도 → Orchestrator → 서브에이전트 위임 → 결과 합성.
3. **툴(스킬) 라우팅**: 스킬 레지스트리에서 도구 호출, 입출력 스키마 검증.
4. **훅 실행**: 모든 도구 호출 전후로 Hook Engine 통과(가드레일·감사).
5. **권한·정책 게이트**: 파괴적/외부전송 액션은 사용자 확인 요구(03b 참조).
6. **데이터 추상화**: DICOMweb/DIMSE, 리포트 DB, AI 모델 게이트웨이를 단일 계층으로 노출.
7. **관측성**: 감사 로그, 추적(trace), 평가 훅(eval harness) 연결.

### 1.2 데이터 액세스 계층 (표준 매핑)
| PACS 동작 | 표준 프로토콜 | 비고 |
|-----------|---------------|------|
| 검사 검색 | QIDO-RS / C-FIND | 워크리스트·검색 |
| 영상 조회 | WADO-RS / C-MOVE·C-GET | 뷰어 표시 |
| 저장·가져오기 | STOW-RS / C-STORE | Import/추가 |
| 전송 | C-STORE | 다른 노드로 전송 |
| 판독문 | FHIR DiagnosticReport / 자체 DB | 리포트 |
| AI 추론 | 모델 게이트웨이(REST/gRPC) | 검출·분류·계측 |

### 1.3 평가 하네스 (Eval Harness)
- **회귀 테스트셋**: 대표 검사(모달리티별) + 기대 동작 시나리오.
- **검증 지표**: 검색 정확도, W/L 적정성, 계측 오차(px/mm), 판독 초안 충실도, 안전(파괴적 액션 차단율).
- **골든 케이스**: 심흉비/Cobb 등 정량 계측은 정답값과 수치 비교(자동).
- 모든 에이전트 변경은 eval 통과를 머지 게이트로 둔다(`03b` Stop 훅과 연계).

## 2. 에이전트(Agents) 로스터

각 에이전트는 **단일 도메인 책임 + 명시적 스킬 집합 + 전용 프롬프트**를 가진다.

### 2.0 Orchestrator Agent (오케스트레이터)
- **역할**: 사용자 자연어 의도 분류 → 적합 서브에이전트로 라우팅, 멀티스텝 작업 조율, 결과 합성.
- **입력**: 자연어 질의 / UI 이벤트 / 현재 컨텍스트.
- **위임 예**: "지난주 흉부CT 미판독 열어서 심흉비 재고 판독 초안" → Worklist→Viewer→Measurement→Reporting 순차 위임.
- **스킬**: `intent_classify`, `route_to_subagent`, `context_summarize`.

### 2.1 Worklist & Search Agent
- **역할**: 검색·필터·정렬·워크리스트 구성.
- **공통기능 매핑**: 1.2 전체.
- **특화**: Filter Property 연산자, 커스터마이즈 워크리스트, prior 자동 정렬.
- **스킬**: `dicom_query`(QIDO/C-FIND), `filter_build`, `worklist_sort`, `search_shortcut_apply`, `nl_to_query`(자연어→검색조건).

### 2.2 Viewer & Layout Agent
- **역할**: 검사 열기(View/Add/Stack), 레이아웃 분할, 행잉 프로토콜, 썸네일.
- **공통기능 매핑**: 1.3.
- **특화**: Advanced Layout, Hanging Protocol 자동 적용, Study With Open.
- **스킬**: `study_open`, `layout_set`, `hanging_protocol_apply`, `thumbnail_arrange`, `series_show`.

### 2.3 Image Manipulation Agent
- **역할**: W/L, Pan, Zoom, 회전/반전, 색상, 필터, 보간.
- **공통기능 매핑**: 1.4.
- **특화**: 자동 최적 W/L(모달리티·부위 인식), Sharpen/Average, 보간법.
- **스킬**: `window_level_set`, `wl_preset_apply`, `auto_window_level`(AI), `pan_zoom`, `rotate_flip`, `color_map`, `image_filter`, `reset_view`.

### 2.4 Measurement & Annotation Agent
- **역할**: 길이/각도/ROI/CTR/Cobb 계측, 주석 작성·편집.
- **공통기능 매핑**: 1.5.
- **특화**: 자동 계측·라벨링(심흉비/척추각), 검출 병변 오버레이 주석.
- **스킬**: `measure_length`, `measure_angle`, `measure_roi`, `measure_ctr`(심흉비), `measure_cobb`, `draw_annotation`, `auto_measure`(AI), `annotation_save`.

### 2.5 Comparison & Sync Agent
- **역할**: prior 비교, Crosslink/Scout/3D Cursor, 상세 동기화, Cine.
- **공통기능 매핑**: 1.3 일부 + 재생.
- **특화**: modality+부위 자동매칭 비교, slice/ratio/mm/px/% 동기화, 변화 하이라이트(AI).
- **스킬**: `prior_match`, `crosslink_sync`, `series_sync`(slice/ratio/mm/px), `cine_play`, `change_highlight`(AI).

### 2.6 3D / Reconstruction Agent
- **역할**: MPR·MIP·VR·Virtual Endoscopy·Segmentation.
- **특화 매핑**: 2.1(INFINITT 강점), 2.3(SonicPACS MPR).
- **스킬**: `mpr_render`, `mip_render`, `volume_render`, `segment_3d`.
- *주: 1차 범위에서는 MPR 우선, 고급 3D는 단계적.*

### 2.7 Reporting Agent
- **역할**: 판독문 작성·템플릿·상태관리·AI 초안.
- **공통기능 매핑**: 1.6.
- **특화**: 음성판독(STT), AI 판독 초안, Merge/Copy/Related 판독, 구조화 리포팅.
- **스킬**: `report_draft`(AI 초안), `template_insert`, `report_save`(Draft/Final), `report_confirm`, `report_suspend`, `report_merge`, `report_pdf`, `voice_transcribe`(STT).

### 2.8 Data Transfer & Lifecycle Agent
- **역할**: 전송·내보내기·가져오기·추가·수정·삭제·CD Burn·익명화.
- **공통기능 매핑**: 1.7.
- **특화**: ISO/ZIP/DICOMDIR+뷰어첨부, JPEG/PNG/PDF→DICOM 변환, 익명화.
- **스킬**: `dicom_send`(C-STORE), `dicom_export`(DCM/ISO/ZIP/DICOMDIR), `dicom_import`, `image_add`, `study_edit`, `study_delete`, `anonymize`, `cd_burn`.
- **⚠ 가드레일 집중 영역**: 삭제·전송·외부내보내기·공유는 모두 확인 훅 필수(03b).

### 2.9 AI Analysis Agent (차별화 핵심)
- **역할**: 검출/분류/정량화 모델 호출, 결과를 주석·판독·우선순위로 환원.
- **스킬**: `ai_detect`(병변), `ai_classify`, `ai_quantify`(CTR/결절크기 등), `ai_triage`(긴급도), `ai_overlay`(마스크 주석화).
- 결과는 항상 **출처·신뢰도 표기** + 사람 확정 단계(human-in-the-loop)를 거친다(03b 정책).

### 2.10 Admin & Audit Agent
- **역할**: 사용자 설정, 단축키, External Link, 감사 로그 질의, 문제해결.
- **공통기능 매핑**: 1.8 + 운영.
- **스킬**: `prefs_set`, `shortcut_config`, `external_link_call`, `audit_query`, `troubleshoot`(서버연결/Pin/검은화면 진단).

## 3. 오케스트레이션 흐름 예시 (시퀀스)

**사용자**: "이 환자 지난 흉부 CT랑 비교해서 결절 변화 있나 보고, 판독 초안 잡아줘"

```
1. Orchestrator   → intent: compare + AI analysis + report
2. Worklist Agent → prior_match(현재 환자, modality=CT, bodypart=CHEST)
3. Viewer Agent   → study_open(현재+prior, Stack/병렬), hanging_protocol_apply
4. Sync Agent     → series_sync(slice position), change_highlight(AI)
5. AI Agent       → ai_detect(결절), ai_quantify(크기), 변화량 계산
6. Measure Agent  → auto_measure → 주석 오버레이
7. Reporting Agent→ report_draft(AI 초안: 결절 위치·크기·변화)
8. [Hook] Stop    → eval 검증 + PHI 점검 + "초안입니다/확정 필요" 표기
9. Orchestrator   → 결과 합성 후 사용자에게 제시(확정은 사람)
```

각 단계의 도구 호출은 **PreToolUse/PostToolUse 훅**(감사·검증·권한)을 통과한다.
