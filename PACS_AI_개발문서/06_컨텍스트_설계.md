# 06. 컨텍스트 설계 (Context Design)

> 컨텍스트 = 에이전트가 매 턴 참조하는 **세션 상태 모델**. 하네스가 유지하고, 프롬프트에 주입하며, 훅이 검증한다.
> 컨텍스트는 (1) **모드 프로파일**(05), (2) **세션·사용자**, (3) **임상 컨텍스트**(환자/검사/시리즈), (4) **뷰 상태**, (5) **작업 상태**로 구성된다.

---

## 1. 컨텍스트 계층 구조

```
AppContext
├── mode          : ProductMode + ModeProfile(05)  ← UI/기능 노출 결정
├── session       : 사용자·권한·로케일·감사세션
├── clinical      : 환자 → 검사 → 시리즈 → 인스턴스 (현재 선택)
├── view          : 레이아웃·W/L·transform·주석·동기화 상태
├── worklist      : 쿼리·결과·정렬·선택
├── task          : 진행 중 멀티스텝 작업·확인대기·AI결과
└── policy        : 가드레일 상태(확인 토큰·PHI 마스킹 규칙)
```

---

## 2. 컨텍스트 스키마 (TypeScript)

```typescript
interface AppContext {
  mode: {
    product: "INFINITT" | "UBPACS_Z" | "SONIC" | "SAINTVIDW";
    profile: ModeProfile;            // 05의 프로파일
    flags: Record<string, boolean | number>;
  };

  session: {
    userId: string;
    role: "radiologist" | "technologist" | "clinician" | "admin";
    locale: string;                  // ko-KR 등
    auditSessionId: string;
    prefs: UserPrefs;                // 툴바/단축키/행잉
  };

  clinical: {
    patient?: { idMasked: string; sexAge?: string };   // PHI는 마스킹 보관
    study?:  { uid: string; modality: string; bodyPart?: string; date: string; reportStatus: "none"|"draft"|"final" };
    priorStudies?: Array<{ uid: string; modality: string; date: string }>;
    series?: { uid: string; count: number };
    instance?: { uid: string; index: number };
  };

  view: {
    layout: { study: string; series: string; image: string };   // "2x2" 등
    openMode: "view"|"addView"|"stackView"|"advanceView"|"keyImageView"|"studyWithOpen";
    wl: { width: number; center: number; preset?: string; auto?: boolean };
    transform: { zoom: number; panX: number; panY: number; rotate: number; flipH: boolean; flipV: boolean };
    colorMap?: string; filter?: { type: string; level: number };
    annotations: Annotation[];
    sync?: { mode: "slice"|"ratio"|"normal"|"mm"|"px"|"percent"|"crosslink"; enabled: boolean };
    cine?: { playing: boolean; fps: number; reverse: boolean };
    hanging?: string;
  };

  worklist: {
    query: { filter: QueryFilter; preview?: string; operators?: string[] };
    results: StudyRow[];
    sort?: { column: string; dir: "asc"|"desc" };
    selected: string[];              // study uids
    shortcuts?: SearchShortcut[];
  };

  task: {
    plan?: Array<{ agent: string; skill: string; args: any; status: "pending"|"done"|"blocked" }>;
    pendingConfirm?: { action: string; args: any; summary: string };  // 확인 대기
    aiResults?: Array<{ skill: string; value: any; confidence: number; model: string; verified?: boolean; labeled: boolean }>;
  };

  policy: {
    confirmTokens: Record<string, boolean>;   // 사용자 확인 기록(per-action)
    phiMask: (s: string) => string;
    destructivePending: boolean;
  };
}
```

## 3. 컨텍스트 ↔ 빌딩블록 상호작용

| 블록 | 읽는 컨텍스트 | 쓰는 컨텍스트 |
|------|---------------|----------------|
| **프롬프트** | mode, session.role, clinical, view, worklist | – (주입만) |
| **스킬** | clinical, view, mode.flags | view, worklist, task.aiResults |
| **훅(Pre)** | policy, mode.flags, task.pendingConfirm | policy.confirmTokens |
| **훅(Post)** | task, clinical | task.aiResults(verified), audit |
| **하네스** | 전체 | session, task.plan, mode |
| **에이전트** | mode.flags(노출기능), clinical, view | view, task |

핵심 규칙:
- 에이전트는 `mode.flags`로 **켜진 기능만** 제안/실행한다(모드별 UI 일치).
- AI 결과는 `task.aiResults`에 `labeled/confidence/verified`와 함께만 적재(03b).
- 확인이 필요한 액션은 `task.pendingConfirm`에 적재 → 사용자 응답으로 `policy.confirmTokens` 갱신 후 실행.

## 4. 컨텍스트 주입(프롬프트 렌더링) 예

```
[MODE] {mode.product} (flags: voiceReport={flags.voiceReport}, ai={flags.ai})
[USER] role={session.role}, locale={session.locale}
[PATIENT] {clinical.patient.idMasked} / study={clinical.study.modality} {clinical.study.bodyPart} {clinical.study.date} ({clinical.study.reportStatus})
[VIEW] layout={view.layout.series}, W/L={view.wl.width}/{view.wl.center}, zoom={view.transform.zoom}
[PRIORS] {clinical.priorStudies.length}건
[PENDING] {task.pendingConfirm?.summary ?? "none"}
```

> 주입 시 **PHI는 항상 마스킹**(`policy.phiMask`). 원본 PHI는 컨텍스트 내부에서도 마스킹 보관, 표시·전송 시 권한 검증.

---

## 5. 컨텍스트 수명주기

```
SessionStart  → mode 로드(05) + session.prefs/행잉 로드 + audit 오픈
검색           → worklist.query/results 갱신
검사 열기      → clinical.study/series + view.layout/openMode 설정 + 행잉 적용
영상 조작      → view.wl/transform/annotations 갱신
AI 분석        → task.aiResults 적재(라벨·검증)
판독           → clinical.study.reportStatus 전이(none→draft→final)
확인 액션      → task.pendingConfirm → policy.confirmTokens → 실행
Stop          → 최종 안전점검 + 컨텍스트 영속화(prefs/주석 저장)
```

---

## 6. 영속화 vs 휘발

| 영속(서버 저장) | 휘발(세션) |
|-----------------|------------|
| user prefs, 행잉, 단축키, 검색 바로가기 | view.transform, pendingConfirm |
| 주석(annotation_save), 판독문 | task.plan, aiResults(미확정) |
| 모드 선택(기관 기본값) | confirmTokens |
| 감사 로그 | 임시 쿼리 미리보기 |

prefs/주석/모드는 **roaming**(어느 워크스테이션에서도 복원) 지원 권장.
