# 03. 스킬 카탈로그 (Skill Catalog)

> 스킬 = 에이전트가 호출하는 **재사용 단위 기능**. 각 스킬은 `이름 / 담당 에이전트 / 트리거 / 입력 / 출력 / 도구(데이터계층) / 가드레일`을 갖는다.
> 표준 스킬 정의 템플릿은 본 문서 §0, 개별 스킬은 §1~§9.

---

## 0. 스킬 정의 템플릿

```yaml
name: measure_ctr                 # kebab/snake, 동사_명사
agent: Measurement & Annotation   # 소속 에이전트
summary: 흉부 정면영상에서 심흉비(CTR) 자동 산출
trigger: "심흉비", "CTR", "심장크기", 흉부 정면 + 계측 요청
inputs:
  - image_ref: 활성 인스턴스(또는 시리즈)
  - mode: auto | manual
outputs:
  - ctr_value: float (심폭/흉곽폭)
  - overlay: 4선(A/A'/B/B') 주석 좌표
  - confidence: 0~1 (auto일 때)
tools: [wado_fetch, ai_quantify, annotation_write]
guardrails: [phi_safe_log, ai_result_label, human_confirm_for_report]
```

각 스킬 카드는 위 필드의 축약형으로 기술한다.

## 1. 검색·워크리스트 스킬 (Worklist & Search Agent)

| 스킬 | 요약 | 입력 → 출력 | 도구 | 가드레일 |
|------|------|-------------|------|----------|
| `dicom_query` | 검사 검색(조건/무조건/기간) | 조건 → study 목록 | QIDO-RS / C-FIND | phi_safe_log |
| `nl_to_query` | 자연어 → 검색조건 변환 | "지난주 흉부CT 미판독" → 필터 | LLM | query_preview_confirm |
| `filter_build` | Filter Property 연산자 적용 | Keyword/`%K%`/`!=` → 쿼리 | – | – |
| `worklist_sort` | 정렬·컬럼 위치 | 컬럼·방향 → 정렬결과 | – | – |
| `search_shortcut_apply` | 등록 검색버튼 실행 | 단축ID → 결과 | – | – |
| `worklist_customize` | 패널 추가/삭제/배치 | 레이아웃 정의 → 저장 | prefs_store | – |

**핵심 동작 규칙**
- `nl_to_query`는 항상 변환된 조건을 **미리보기 후 실행**(질의 투명성).
- 날짜 토큰: 없음/오늘/어제/지난주(7일)/지난달(31일)/범위 — SonicPACS 사양 채택.

## 9. AI 분석 스킬 (AI Analysis Agent) — 차별화

| 스킬 | 요약 | 출력 | 가드레일 |
|------|------|------|----------|
| `ai_detect` | 병변/이상 검출 | bbox/mask + confidence | ai_result_label |
| `ai_classify` | 소견 분류 | label + prob | ai_result_label |
| `ai_quantify` | 정량화(CTR/결절크기/체적) | 수치 | numeric_verify |
| `ai_triage` | 긴급도 분류·워크리스트 우선순위 | priority score | ai_result_label |
| `ai_overlay` | 모델 결과 → 주석 레이어 변환 | annotation set | human_confirm_for_report |

**AI 스킬 공통 규칙**
1. 출력에는 **모델 버전·신뢰도** 메타데이터 필수.
2. 판독문/임상결정에 반영하기 전 **사람 확정(human-in-the-loop)**.
3. 정량 결과는 `numeric_verify` 훅으로 재계산 검증.
