# UBPACS-Z(유비팍스 제트) 기능 분석 · 개발 지침서

> **목적**: PACS 뷰어 제품 *UBPACS-Z* 의 「기초사용자교육 필수편」 영상을 분석하여, 자체 PACS AI(saintvidw pacs ai) 개발 시 참고·벤치마킹할 기능 단위 지침으로 정리한 문서.
> **출처**: YouTube 「0_기초사용자교육필수편_유비팍스제트(UBPACS-Z)」 (약 29분, 채널: UBPACS-Z_유비팍스제트)
> **분석 기반**: 영상 자동생성 자막(한국어 ASR)을 해석·재구성. 일부 음성인식 오류가 있어 기능 의도 중심으로 정리했으며, 정확한 단축키·메뉴 명칭은 실제 제품 확인이 필요함.
> **영상 성격**: 신규 납품·설치 직후 현장에서 "반드시 교육해야 할" 기본 사용법 가이드. 따라서 본 문서의 기능 목록 ≒ PACS 뷰어가 갖춰야 할 최소 핵심 기능 집합으로 볼 수 있음.

## 1. 로그인 · 인증

**기능 개요**
메인 진입 시 ID/PW 기반 로그인. 반복 사용자를 위한 자동 로그인 및 로그인 직후 뷰어 자동 전환을 지원.

**세부 요구사항**
- ID/비밀번호 입력 폼 제공.
- "자동 로그인" 체크박스: 체크 시 이후 접속에서 자격증명 재입력 없이 자동 로그인.
- "로그인 후 바로 뷰어로 진입" 옵션: 체크 시 로그인 직후 워크리스트가 아니라 뷰어 화면으로 바로 이동.
- 최초 로그인 후 권한·세션 유지.

**벤치마킹 포인트 (AI PACS)**
- SSO/OAuth, 병원 EMR 연동 인증(HL7/FHIR 토큰) 고려.
- "자동 로그인"은 의료정보 보안상 화면 잠금·세션 타임아웃·기기 신뢰등록과 함께 설계.

## 2. 메인 화면 · 기본 레이아웃

**기능 개요**
로그인 직후 표시되는 디폴트 작업 화면. 대부분의 사용자가 동일한 기본 레이아웃을 보게 되며, 영역별로 워크리스트·검색·미리보기·과거검사·관련 패널이 배치됨.

**기본 화면 영역 구성**
- **좌측 상단**: 워크리스트 / 검색조건(쿼리) 영역.
- **좌측 중단**: 검색 결과 리스트.
- **좌측·하단**: 미리보기(썸네일) 창 — 선택한 검사의 시리즈/이미지 미리보기.
- **가운데**: 선택 환자의 과거 검사 이력(리스트).
- **우측**: 선택 검사 관련 부가 UI/패널.

**세부 요구사항**
- 디폴트 레이아웃이 항상 일관되게 제공되어야 함(현장 교육 표준화 목적).
- 패널이 비어 있거나 사라진 경우, 영역에서 **우클릭 → 복구**로 재표시 가능.

**벤치마킹 포인트 (AI PACS)**
- 동일 디폴트 레이아웃 제공 + 역할(판독의/기사/임상의)별 프리셋 레이아웃 추가.
- AI 결과(병변 검출, 우선순위 분류)를 표시할 **전용 패널 영역**을 기본 레이아웃에 선반영.

## 3. 화면구성(레이아웃 커스터마이징)

**기능 개요**
사용자가 직접 패널을 추가·이동·배치하여 자신만의 작업 화면을 구성. "화면구성" 탭에서 관리.

**세부 요구사항**
- "화면구성" 탭/메뉴 진입 → 편집 모드.
- 빈 공간에서 **우클릭 → 패널 추가** 메뉴로 원하는 패널 삽입.
- 패널 상단(타이틀 바)을 잡고 **드래그하여 원하는 위치로 이동/도킹**.
- 구성 완료 후 저장, 잘못 건드린 경우 우클릭으로 초기·복구 가능.
- 썸네일(미리보기) 영역의 크기 축소/확대 조절.

**벤치마킹 포인트 (AI PACS)**
- 드래그&드롭 도킹은 표준 UX. 레이아웃 프리셋 저장/공유, 멀티모니터 배치 저장 지원.
- 사용자별 레이아웃을 서버에 저장하여 어느 워크스테이션에서도 동일 환경 복원(roaming profile).

---

## 4. 환경설정(Preferences)

**기능 개요**
뷰어 동작과 표시 옵션을 제어. 특히 이미지 편집(추가/삭제) 기능은 기본 미노출 상태이며 환경설정에서 활성화해야 함.

**세부 요구사항**
- **EDIT 탭(이미지 추가/삭제)**: 기본 레이아웃에는 없을 수 있음. 환경설정에서 EDIT 탭을 **활성화/추가**해야 함.
  - 용도: 이후 검사 이미지를 **삭제하거나 추가**할 때 반드시 필요한 탭 → 설치 시 미리 추가해 두도록 교육.
- **툴팁 표시 on/off**: 아이콘에 마우스 오버 시 설명(풍선도움말)이 나오지 않으면, 빈 공간 **우클릭 → "툴팁 표시"** 활성화.
- **상태값(state) 설정**: 특정 값을 조정하면 동일 작업(예: 판독 항목)을 **연속해서 수정**할 수 있는 모드로 전환.

**벤치마킹 포인트 (AI PACS)**
- 이미지 추가/삭제는 감사로그(audit trail)·권한제어 필수(의료영상 무결성).
- 기본 노출 여부를 관리자 정책으로 제어(역할 기반 기능 노출).

## 5. 판독(Reporting) 기능

**기능 개요**
워크스테이션 내에서 판독문을 작성. 판독 패널은 기본 화면에 없으므로 화면구성에서 추가하며, 표지판독(템플릿)과 단축키 등록으로 반복 입력을 가속.

**세부 요구사항**
- **판독판 패널 추가**: 기본 화면에는 판독 영역이 없음 → "화면구성 → 구축(추가) → 레퍼런스" 패널을 클릭해 띄우고, 상단을 드래그하여 원하는 위치에 배치.
- **표지판독(판독 템플릿)**:
  - 신규 등록 메뉴에서 정형 문구(예: "정상") 등록.
  - 등록된 항목은 리스트에서 선택·재사용.
- **판독 단축키 등록**:
  - 단축키 F1~F8에 정형 판독문을 매핑.
  - **Ctrl / Shift 조합**으로 확장하여 **최대 24개**(예: F1~F8 × 3세트)까지 등록 가능.
  - 등록 후 저장하면 단축키로 즉시 해당 판독문 입력.
- **판독문 입력 방법**: 등록된 단축키 입력, 또는 항목 **더블클릭**으로 판독판에 삽입.
- 키 입력 + 엔터로 항목 검색·적용.

**벤치마킹 포인트 (AI PACS)**
- 템플릿/매크로/단축키는 판독 생산성의 핵심. **구조화 리포팅(structured reporting)** + 음성인식(STT) + AI 초안 자동생성으로 확장.
- AI가 검출한 소견을 판독문 초안으로 자동 채우고, 판독의가 단축키로 수정/확정하는 워크플로우 설계.

## 6. 이미지 뷰어 조작(Manipulation)

**기능 개요**
키보드 단축키 중심의 확대·이동·윈도우레벨·주석 조작. 모드 전환과 초기화(리셋) 흐름이 명확.

**세부 요구사항 (단축키 — 영상 기준, 실제 매핑 확인 필요)**
- **Z 키 = 확대(Zoom)**: 누르면 커서가 돋보기 모양으로 변경, 드래그/조작으로 확대.
- **R2(또는 지정 키) = 이동(Pan)**: 확대된 이미지를 상하/좌우로 이동하여 다른 부위 관찰.
- **윈도우 레벨(W/L) 조정**: 밝기/대비 조정.
- **ESC**: 확대·주석 등 모드에서 **기본 선택 커서/기본 주석 아이콘으로 복귀**.
- **주석(Annotation)**:
  - 기본 주석 도구 제공(기타 고급 주석은 별도 영상에서 설명).
  - **C 키** 등 주석 관련 단축키 사용.
  - **우클릭 → 리셋**: 적용한 주석·조정을 디폴트로 초기화(모두 삭제).

**벤치마킹 포인트 (AI PACS)**
- 단축키 매핑을 사용자 커스터마이즈 가능하게(표준 단축키 프리셋 + 사용자 정의).
- 마우스 휠/제스처 기반 W/L·스크롤, 프리셋 윈도우(폐/뼈/연부조직 등) 제공.
- AI 오버레이(병변 마스크, 측정 자동화)를 주석 레이어로 토글 표시.

## 7. 출력 · 인쇄

**기능 개요**
이미지 및 판독문 출력. 출력 대상(이미지 전용 / 판독문 포함)을 프린터 옵션으로 선택.

**세부 요구사항**
- 인쇄 아이콘 클릭 → 인쇄 옵션 창(디폴트 옵션으로 표시).
- **이미지 프린터(기본값)**: 이미지만 출력.
- **페이퍼 프린터로 변경**: 판독문이 함께 출력됨.
- 판독문 포함 여부를 출력 단계에서 명확히 선택하도록 교육.

**벤치마킹 포인트 (AI PACS)**
- DICOM 프린터(필름) / 일반 프린터 / PDF 내보내기 분리 지원.
- 판독문+키이미지 합성 리포트, 환자 제공용 안전한 PDF/QR 공유.

## 8. 운영 · 서비스 관점(설치/교육 정책)

**기능 개요 (제품이 아닌 운영 지침)**
영상은 "납품·설치 직후 반드시 교육해야 할 항목"을 강조. 사후 서비스 부담을 줄이기 위한 표준 교육 절차의 성격.

**핵심 메시지**
- 설치 시 EDIT 탭 추가, 기본 화면구성, 판독 단축키 등록 등 **초기 셋업을 표준화**하여 교육.
- 메뉴·기능을 고객에게 충분히 설명해두면 이후 원격/방문 서비스가 수월해짐.

**벤치마킹 포인트 (AI PACS)**
- 설치 마법사(초기 셋업 자동화), 인앱 온보딩 투어, 역할별 기본 프리셋 자동 적용으로 "교육 의존도"를 낮추는 방향이 차별화 포인트.

## 9. 개발 체크리스트(최소 기능 세트)

- [ ] ID/PW 로그인 + 자동 로그인 + 로그인 후 뷰어 자동 진입 옵션
- [ ] 표준 디폴트 레이아웃(워크리스트/검색/썸네일/과거검사/우측 패널)
- [ ] 우클릭 기반 패널 추가·복구, 드래그 도킹, 레이아웃 저장

- [ ] 판독판 패널(레퍼런스) 추가 + 표지판독 템플릿
- [ ] 판독 단축키 등록(F1~F8, Ctrl/Shift 확장 최대 24개) + 더블클릭 삽입
- [ ] 이미지 조작: 확대(Z)/이동(R2)/W·L/주석(C)/ESC 복귀/우클릭 리셋
- [ ] 인쇄: 이미지 프린터 ↔ 페이퍼 프린터(판독문 포함) 선택
- [ ] (운영) 초기 셋업 표준화·온보딩

## 10. 검증·확인이 필요한 항목(자막 해석 한계)

다음 항목은 ASR 자막 기반 추정으로, **실제 제품/문서로 정확히 확인** 후 확정 권장:
- 단축키 실제 매핑(Z=확대, R2=이동, C=주석 등 명칭/키값).
- "표지판독", "레퍼런스", "상태값", "메디텔/EDIT 탭" 등 정확한 메뉴 명칭.
- 단축키 최대 등록 개수(24개) 및 조합 규칙.
- 자동 로그인/뷰어 자동 진입 옵션의 정확한 위치.
