# 05. 제품별 모드 디자인 분석 (Product Mode Design)

> 목표: INFINITT / UBPACS-Z(ZeTTA) / SonicPACS 각 제품의 **Worklist·Viewer 디자인을 코딩 가능한 형태로 분석**하고, 하나의 코드베이스에서 **제품을 "모드"로 전환**할 수 있는 설정 구조를 정의한다.
> 핵심 아이디어: **공통 코어(Common Core)** + **제품별 모드 프로파일(Mode Profile)** = 제품 UI/동작 재현. saintvidw 기본 모드는 공통 코어 + AI 확장.

---

## 1. 설계 원칙 — Core + Mode Profile

```
                ┌─────────────── Common Core ───────────────┐
                │  Worklist Engine · Viewer Engine · Toolbar │
                │  Layout Engine · Report Engine · DataLayer │
                └───────────────────┬───────────────────────┘
                                    │ reads
                ┌───────────────────▼───────────────────────┐
                │            Mode Profile (JSON)             │
                │  worklistPanels[] · viewerOpenModes[]      │
                │  toolbarGroups[] · layoutOptions[]         │
                │  featureFlags{} · shortcuts{} · theme{}    │
                └────────────────────────────────────────────┘
   Modes: INFINITT | UBPACS_Z | SONIC | SAINTVIDW(default, +AI)
```

- 코어는 **모든 기능을 구현**하되, 각 기능의 **노출/배치/기본값**은 Mode Profile이 결정.
- 모드 전환 = Profile 교체(런타임). 기능 자체는 featureFlag로 on/off.

### 2.2 제품별 Worklist 배치 특성
- **INFINITT**: 좌측 검색+워크리스트, 하단 미리보기, 우측 관련검사/리포트. Step 기반 단순 흐름.
- **UBPACS-Z**: **최대 10개 컴포넌트 자유 배치**(Search Wizard / Study List / Related 1·2 / Order / Reference / Comment / Memo / Thumbnail / Report). 사용자 설정 추가·삭제.
- **SonicPACS**: 검사목록 중심 + **모달리티/환자 목록 필터**, 강한 상태표시줄(용량/백업), 썸네일바 위치(좌/우/상) 전환.

## 5. 모드 전환 컴포넌트 (코드 골격)

```typescript
// modeRegistry.ts
export type ProductMode = "INFINITT" | "UBPACS_Z" | "SONIC" | "SAINTVIDW";

export interface ModeProfile {
  worklist: WorklistProfile;
  viewer:   ViewerProfile;
  toolbarGroups: ToolbarGroup[];
  featureFlags: Record<string, boolean | number>;
  shortcuts: Record<string, string>;
  theme?: ThemeTokens;
}

const REGISTRY: Record<ProductMode, ModeProfile> = {
  INFINITT:  loadProfile("modes/infinitt"),
  UBPACS_Z:  loadProfile("modes/ubpacs_z"),
  SONIC:     loadProfile("modes/sonic"),
  SAINTVIDW: loadProfile("modes/saintvidw"),
};

export function applyMode(mode: ProductMode, ctx: AppContext) {
  const profile = REGISTRY[mode];
  ctx.worklist.configure(profile.worklist);   // 패널/슬롯/검색
  ctx.viewer.configure(profile.viewer);        // openModes/layout/sync
  ctx.toolbar.render(profile.toolbarGroups);   // 도구 그룹 노출
  ctx.flags = profile.featureFlags;            // 기능 on/off
  ctx.shortcuts.load(profile.shortcuts);
  ctx.theme.apply(profile.theme);
  ctx.emit("mode:changed", mode);
}
```

- **featureFlag 게이팅**: 모든 스킬 호출 전 `ctx.flags[skill.flag] ?? false` 확인(하네스 Tool Router).
- **에이전트 연동**: 모드 프로파일은 컨텍스트(`06`)의 `mode`·`flags`로 주입 → 에이전트가 노출 기능만 제안.

## 6. 코드 구현을 위한 매핑 요약

| 코딩 대상 | 입력(분석 산출) | 산출 코드 |
|-----------|----------------|-----------|
| Worklist UI | §2 패널 인벤토리·Profile JSON | `WorklistEngine.configure(profile)` |
| Viewer 모드 | §3 openMode/layout/toolbar JSON | `ViewerEngine.configure(profile)` |
| 도구 노출 | §3.3 toolbarGroups | `Toolbar.render(groups)` |
| 기능 on/off | featureFlags | Tool Router 게이트 |
| 모드 전환 | §5 registry | `applyMode()` |

데이터 스키마(타입 정의)와 단계별 plan은 `07_구현스키마_PLAN.md`.
